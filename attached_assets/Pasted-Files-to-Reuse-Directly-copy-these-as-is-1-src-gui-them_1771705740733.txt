Map Selection Code to Reuse (from src/modules/bot_engine.py)
This is the core logic that needs to be adapted. Here's each state handler explained:

_handle_selecting_map_zone (lines ~280-324)
What it does: Clicks the Glacial Abyss button to open the card grid

Gets glacial_abyss_click coordinates from config (default: [530, 599])
Calls _game_click(x, y) which converts game coords → screen coords then clicks
V1 verification used image recognition to check if the card grid opened (is_card_selection_screen). V2 needs to replace this — either read memory to know what UI is open, or use a simple timed delay
_handle_selecting_card (lines ~326-398)
What it does: Picks a map card from the hex grid

Gets card_positions dict from config — 12 maps with their top vertex coordinates
V1 used find_active_cards() which scanned for glow pixels above each card to detect which are active and their rarity. This is the IMAGE RECOGNITION part that V2 needs to replace.
After detecting active cards, picks the rarest one (or first available)
Clicks at (card_x, card_y + 58) — the stored position is the TOP VERTEX, offset +58px down to hit the card center
After clicking, checks if a detail panel opened on the right side
If panel didn't open, retries the same card (stores as _pending_card)
V2 approach: If memory reading can tell you which cards are active and their rarity, great. If not, you may still need some visual detection here, OR just click all known positions and see which one opens a panel.

_handle_card_detail (lines ~400-444)
What it does: Handles the detail panel after clicking a card

Looks for button text at bottom-right: either "Next" or "Open Portal"
V1 used OCR (detect_button_text) to read the button. V2 could read UI state from memory instead.
If "Next" → clicks it (transitions to affix screen)
If "Open Portal" → skips to affix stage (already past "Next")
Button position: next_button at [1758, 998]
_handle_adding_affixes (lines ~446-491)
What it does: Clicks "Add Affix" button repeatedly

Clicks add_affix_button at [120, 819] up to affix_count times (default: 5)
Optionally detects red (dangerous) affixes and resets if found
reset_affix_button at [380, 819]
V1 used OCR to read affix text and drop quantity. V2 could read from memory.
After all affixes added, moves to OPENING_PORTAL state
_handle_opening_portal (lines ~493+)
What it does: Clicks "Open Portal" button

open_portal_button at [1758, 998] (same position as Next button — they swap in place)
After clicking, the map device UI closes and a portal spawns in hideout
Bot then needs to walk to the portal and press F
_game_click helper (line ~127)
What it does: Converts game coordinates to screen coordinates and clicks

def _game_click(self, game_x, game_y, delay=0.3):
    screen_x, screen_y = self.window.game_to_screen_coords(game_x, game_y)
    self.input.click(screen_x, screen_y)
    time.sleep(delay)

What V2 Needs to Replace
The only parts that used image recognition in the map selection flow were:

Detecting which cards are active (find_active_cards) — glow pixel scanning
Reading button text (detect_button_text) — OCR for "Next" vs "Open Portal"
Detecting right panel opened (detect_right_panel_open) — visual check
