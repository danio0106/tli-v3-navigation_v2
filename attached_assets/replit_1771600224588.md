# Torchlight Infinite Bot

## Overview
This project is a Python-based automation tool for the game Torchlight Infinite. Its primary purpose is to automate the complete map cycle, from selecting and starting new maps to navigating them, picking up loot, and returning to the hideout. It's designed to be a fully AI-generated, clean, and user-friendly application, targeting a modern dark-themed GUI. The bot handles in-game navigation for an auto-bomber build, ensuring efficient map clearing and resource gathering.

## User Preferences
- University project - code must be fully AI-generated, clean, and usable by end users
- Modern, clean GUI design
- Dark theme with accent colors
- Game runs at 1920x1080 windowed mode on 2560x1440 desktop
- Character build: auto-bomber (skills auto-cast, no manual skill usage)
- Minimap is in top-LEFT corner (not top-right)
- "F" key for all interactions (portals, map device)
- "E" key for loot pickup (spam during map)
- Location text "Hideout - Ember's Rest" in top-left indicates hideout
- **NEVER suggest unreliable solutions** (color blob detection, HSV averaging for positions, etc.)
- **Always ask user for coordinates** before pulling color data — never guess positions from screenshots
- Player position is **ALWAYS hardcoded at (263, 248)** game-window coordinates (paint coords 264,279 minus title border 1,31). NO detection needed.
- Window title border offset: (1, 31) — subtract from Paint coordinates to get game-window coordinates

## Critical Rules
- **NEVER right-click after initial map start.** Character follows cursor automatically. To STOP: move cursor to character center (959, 489 client area).
- **Player arrow is always at the center of the minimap.** The minimap scrolls around the player, not vice versa. No player detection (gold/color/template) is needed.
- **Player arrow color varies**: Sometimes golden (H:15-45), sometimes pink/red (H:0-10) depending on map. This is irrelevant since we don't detect it.
- **All coordinate reading must be verified** — automated calibration tool is preferred over manual guessing.

## System Architecture
The bot is built with a clear modular structure, primarily in Python, using `customtkinter` for its GUI.
- **Core Logic**: A state machine (`bot_engine.py`) manages the bot's workflow, transitioning between states like `IN_HIDEOUT`, `SELECTING_MAP_ZONE`, `RUNNING_MAP`, and `RETURNING_HIDEOUT`.
- **GUI**: A modern dark-themed GUI is implemented using `customtkinter`, with defined color palettes and font definitions in `theme.py`.
- **Input Control**: `input_controller.py` handles mouse and keyboard automation.
- **Screen Analysis**: `screen_reader.py` performs screen capturing, image analysis, and game state detection using `opencv-python-headless`. Key detection methods include:
    - **Color-based Detection**: Utilizes HSV color analysis for elements like hideout detection (orange text), card activity, and rarity scoring.
    - **OCR Detection**: Employs `pytesseract` for fuzzy matching of text, such as map names, button states ("Next", "Open Portal"), and affix limits.
    - **Template Matching**: For identifying interact icons, portal icons, sandlord platforms.
- **Calibration Tool**: `calibration_tool.py` auto-detects minimap boundaries using edge/contour detection from a live screenshot. Saves coordinates to `~/.torchlight_bot/calibration_data.json`. Generates annotated preview for visual confirmation. GUI "Calibrate" button triggers it.
- **Game Window Management**: `window_manager.py` detects the game window and maps coordinates.
- **Map Navigation**: `map_navigator.py` is the core navigation engine, incorporating:
    - **Movement Model**: Character moves towards the cursor with a single right-click, auto-attacking. Subsequent movements are solely cursor-driven.
    - **Minimap Scanning**: A high-speed background thread (`MinimapScanner`) captures the minimap. Player position is ALWAYS the minimap center (hardcoded). Wall avoidance uses ray casting (8px lookahead).
    - **Absolute Coordinate System**: Frame-to-frame minimap offset tracking using `cv2.phaseCorrelate` on terrain grayscale images. Accumulates (dx, dy) shifts to build absolute position. Waypoints are recorded and navigated using absolute coordinates, eliminating minimap scrolling issues.
    - **Teleport Detection**: Large offset jumps (>60px) between frames are logged as teleports. Portal gaps between waypoints (>80px distance) are auto-detected during waypoint loading.
    - **Debug Overlay**: Transparent overlay window (`debug_overlay.py`) positioned over game minimap showing absolute position (green), target waypoint (blue), heading line, wall hits (red), event markers (yellow/orange/purple), waypoint progress.
    - **Debug Logging**: Verbose per-frame logging toggle shows absolute position, heading, target waypoint, distance, wall info every 3 seconds.
    - **Waypoint Navigation**: Uses absolute coordinates from recorded paths. The navigator calculates heading towards the next waypoint using absolute position vs absolute waypoint target.
    - **Event Handling**: Detects and handles in-map events like carjack trucks and sandlord platforms via minimap color/contour detection with state machine (approach → interact → wait → resume).
    - **Portal-Aware Navigation**: Continuously scans for portal interact prompt (F icon) during navigation. When detected, presses F to enter portal, waits for teleport, then resumes waypoint navigation. No distance-based detection — purely visual prompt scanning.
    - **Path Recording**: GUI "Record Path" button captures absolute positions at intervals during manual play, simplifies to up to 80 waypoints via angle/distance filtering, saves to JSON. Recorded paths override hardcoded MAP_CONFIGS.
    - **Smart Start**: On bot start, detects if already in a map (not in hideout) and skips map selection, going straight to navigation.
    - **Heading Smoothing**: 70/30 blend toward target heading reduces cursor jitter. Emergency wall avoidance uses full redirect.
    - **Stuck Recovery**: 15s no-waypoint-progress detection triggers alternating ±90° escape angles.
    - **Carjack Star HUD**: Monitors game region (1798, 185, 74x43) for white "5" text (complete) vs green countdown (in progress).
    - **Exit Portal Detection**: After all waypoints completed, automatically searches minimap for exit portal using template matching (portal icon template) with HSV color fallback. Steers toward portal while spamming F key. 30s search timeout.
- **Configuration**: Settings are persisted using JSON via `config_manager.py`.
- **Coordinate System**: All UI button positions and detection regions are defined using configurable game-relative coordinates for 1920x1080 resolution. Minimap coordinates come from calibration tool (`calibration_data.json`).
- **Card Rarity System**: Detects card rarities (Rainbow, Orange/Golden, Purple, Blue) based on hue variance, saturation, and value in specific regions above the card hex.
- **Thread Safety**: The bot engine runs in a daemon thread, with GUI updates managed via `tkinter`'s `after()` method.
- **Human-like Interaction**: Mouse movements and clicks are designed to mimic human interaction, with pauses and `mouseDown/mouseUp` events.

## External Dependencies
- `customtkinter`: For the graphical user interface.
- `pyautogui`: For mouse and keyboard control.
- `opencv-python-headless`: For image analysis and screen detection.
- `numpy`: For numerical operations, especially with image data.
- `Pillow`: For general image handling.
- `pytesseract`: For Optical Character Recognition (OCR), requires Tesseract OCR to be installed on the system.
- `mss`: For fast screen capturing.
